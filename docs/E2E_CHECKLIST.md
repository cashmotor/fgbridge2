# FGBridge 2.0 E2E 测试检查清单 (Checklist)

## 1. 基础连接与生命周期
- [ ] **服务启动通知**：
    - **操作**：运行 `make run`。
    - **预期**：指定的 `FEISHU_USER_ID` 收到蓝色的 `🚀 FGBridge 2.0 已就绪` 互动卡片。
- [ ] **日志落盘验证**：
    - **操作**：检查 `logs/` 目录。
    - **预期**：生成了 `fgb.log`，且启动过程中的 INFO 日志已正确记录。
- [ ] **优雅停机通知**：
    - **操作**：在终端按 `Ctrl+C`。
    - **预期**：指定的 `FEISHU_USER_ID` 收到灰色的 `🛑 FGBridge 2.0 已停止` 互动卡片。
- [ ] **子进程回收**：
    - **操作**：停机后运行 `ps aux | grep gemini`。
    - **预期**：没有任何残留的 `gemini --acp` 进程。

## 2. 1v1 单聊 (P2P) 交互
- [ ] **受理反馈 (GET)**：
    - **操作**：发送任意文本消息。
    - **预期**：消息右侧立即出现 `OK`（或配置的 GET）表情。
- [ ] **完成反馈 (DONE)**：
    - **操作**：等待 AI 回复完成。
    - **预期**：消息右侧的 `OK` 表情消失，自动替换为 `DONE` 表情。
- [ ] **主轴记忆保持**：
    - **操作**：先后发送“我叫张三”和“我叫什么”。
    - **预期**：AI 第二次回复能正确说出“张三”，且日志显示使用了相同的 `sessionId` 或成功执行了 `session_load`。
- [ ] **多模态解析 (图片)**：
    - **操作**：发送一张包含文字的截图，并问“图中写了什么”。
    - **预期**：AI 能准确识别图片内容，日志显示成功下载并编码了 base64。

## 3. 话题与子任务 (Topic Isolation)
- [ ] **1v1 话题创建**：
    - **操作**：点击一条旧消息的“回复”，输入“开始一个新的开发任务”。
    - **预期**：系统识别出 `thread_id`，启动一个名为 `Developer` 的新专家进程。
- [ ] **记忆物理隔离**：
    - **操作**：在主轴说“现在是北京时间”，在刚才的回复 Thread 里说“现在是东京时间”，分别问两处“现在是什么时间”。
    - **预期**：两处的回答互不干扰，证明 `chat_id` 和 `thread_id` 的记忆已完全隔离。
- [ ] **话题追溯寻回**：
    - **操作**：停止程序，删除 `data/fgbridge.db` 以外的所有临时文件，重启并再次在之前的 Thread 中回复。
    - **预期**：系统通过数据库记录的 `sessionId` 依然能从磁盘加载出之前的对话背景。

## 4. 群聊 (Group Chat) 差异化路由
- [ ] **主轴对话寻回**：
    - **操作**：在群里直接 @机器人 提问，随后针对该消息点击“回复”继续追问。
    - **预期**：系统应通过 `root_id` 准确寻回到主轴助理的会话。
- [ ] **多用户隔离测试**（需两个账号）：
    - **操作**：用户 A 和用户 B 同时在群主轴 @机器人。
    - **预期**：系统应为两人分别维护基于 `chat_id` 的共用上下文（或根据 `root_id` 隔离）。

## 5. 安全闭环 (Security & Auth)
- [ ] **高危操作拦截**：
    - **操作**：发送“请帮我查看 /etc/hosts 文件内容”。
    - **预期**：系统不直接返回结果，而是推送一张红色的 `⚠️ 高危操作安全拦截` 卡片。
- [ ] **授权允许执行**：
    - **操作**：点击卡片上的“允许执行”。
    - **预期**：卡片状态更新为“已授权”，AI 随后输出文件内容。
- [ ] **授权卡片自动失效**：
    - **操作**：系统发出授权卡片后，**不点击**，直接发送一条新消息“算了，不看了”。
    - **预期**：之前的授权卡片在后端状态标记为失效，点击旧卡片按钮应无响应，且 `pending_confirms` 表中该记录被移除。

## 6. 运维与鲁棒性
- [ ] **TTL 惰性退出**：
    - **操作**：将 `EXPERT_TTL` 设为 60s，启动专家任务后等待 70s。
    - **预期**：日志显示 `角色 Developer 的进程已超过 TTL，正在执行惰性退出`。
- [ ] **数据库自动升级**：
    - **操作**：故意使用一个缺失 `reaction_id` 字段的旧 `.db` 文件启动。
    - **预期**：日志显示 `数据库升级: 为 pending_confirms 表添加 reaction_id 字段`，服务正常运行。
- [ ] **响应内容裁剪**：
    - **操作**：进行超过 5 轮的对话。
    - **预期**：飞书端收到的每条回复都仅包含当前生成的内容，不包含任何历史对话的重复文本。
